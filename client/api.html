Run a Prediction:

  
  ===
      Run a prediction
  ===

  Definition:
  
  POST /predict
  

  Example Request Body: 

  
                                
post_data = {
    'userId': XXXXXXXXXX,
    'insert': true,
    'accel': [
        {
            'userId': XXXXXXXXXX // user id code
            'x': 1, // floating point value
            'y': 2, // floating point value
            'z': 3, // floating point value
            'timestamp': XXXXXXXXX // (timestamp in ms)
        },
        ...
    ],
    'gyro': [
        {
            'userId': XXXXXXXXXX // user id code
            'x': 1, // floating point value
            'y': 2, // floating point value
            'z': 3, // floating point value
            'timestamp': XXXXXXXXX // (timestamp in ms)
        },
        ...
    ]
}

  
  
  Example Response: 
  // Successful DB response.
  
  {
      inserted: 2,
      prediction: stimming // 'stimming' or other classified activity
  }

      {
        'Name': 'Stim Entity',
        'ID': X,
          ...
      },
      ...
  ]

Get Fitbit Data:

  
  ===
      Get Accelerometer and Gyroscope Data
  ===

  Definition:

  
  GET /accel
  

  Example Request Body: 
  
      {
          userId: 'XXXXXXXXX',// userId of desired user.
          startTime: 'XXXXXXXXX', // start time of accelerometer data events in epoch milliseconds
          endTime: 'XXXXXXXXX'// end time of accelerometer data events in epoch milliseconds
      }
  

  Example Response:
  // Successful DB response.
  
  
 {
      'data': [
          {
            "userId": XXX,
            "x": 1,
            "y": 2,
            "z": 3,
            "timestamp": XXX
          },
        ...
      ]
  }
  

  

 Definition:
  
  GET /gyro
  

  Example Request Body: 
  
      {
          userId: 'XXXXXXXXX',// userId of desired user.
          startTime: 'XXXXXXXXX', // start time of accelerometer data events in epoch milliseconds
          endTime: 'XXXXXXXXX'// end time of accelerometer data events in epoch milliseconds
      }
  

  Example Response:
  // Successful DB response.
  
  
 {
      'data': [
          {
            "userId": XXX,
            "x": 1,
            "y": 2,
            "z": 3,
            "timestamp": XXX
          },
        ...
      ]
  }
  

      Note that the gyro data response takes the exact same form as the accelerometer - aside from differences in units.
  
                        

                        

                            

                                
Get Optimal Schedule for the Day:

                            

                            
  
  ===
  Get Optimal Schedule for day
  ===

  Definition:
  
  POST /api/schedule
  

  Example Request Body:
  
  {
      startPortId: 0, // origin and return port for vehicles
      numVehicles: 2, // number of vehicles starting at port
      jobDate: '4-14-2018' // jobDate for optimal schedule computation
  }
  

  Example Response:
  
  {
      cost: X, // optimal cost for schedule
      routes:[[],[]], // nodes for each vehicle to visit (index i = route for truck i)
      times:[[],[]] // times that truck i can acceptablely arrive/leave at each route node,
  }
